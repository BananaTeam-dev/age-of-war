# Coding Style Guide

Files are written in `snake_case`, e.g. `my_cool_file.gd`
Classes are written in `PascalScale`, e.g. `MyClass`

### GDScript Documentation

Use one hashtag `#` for (used for devs):
- intent ("why"), constraints
- non-obvious logic
- TODO/FIXME with context

Use two hashtag `##` for (used for Godot descriptions when hovering):
- script/class description (top-of-file)
- exported variables you want tooltips for
